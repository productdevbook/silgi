Katman supports three wire formats. The server picks the format based on the client's `Accept` header — zero configuration.

## Comparison

| Format | Size | Speed | Rich Types | Binary |
|---|---|---|---|---|
| **JSON** | Baseline | Fastest | No | No |
| **MessagePack** | 30% smaller | Fast | Date only | Yes |
| **devalue** | ~Same as JSON | Fast | All | No |

## JSON (default)

Standard JSON. Fastest encode/decode. Universal compatibility.

```ts
// Nothing to configure — it's the default
const link = createLink({ url: "http://localhost:3000" })
```

## MessagePack

Binary protocol — 30% smaller payloads. No competitor offers this.

<Tabs items={['Client', 'Direct']}>
  <Tab value="Client">
  ```ts
  const link = createLink({
    url: "http://localhost:3000",
    binary: true, // [!code highlight]
  })
  ```
  </Tab>
  <Tab value="Direct">
  ```ts
  import { encode, decode } from "katman/msgpack"

  const buf = encode({ name: "Alice", joined: new Date() })
  const data = decode(buf) // Date preserved!
  ```
  </Tab>
</Tabs>

## devalue

Handles every JavaScript type — Date, Map, Set, BigInt, RegExp, undefined, circular references:

```ts
import { encode, decode } from "katman/devalue"

const data = {
  users: new Map([["alice", { id: 1 }]]),
  tags: new Set(["admin"]),
  count: 9007199254740993n,
  created: new Date(),
}

const str = encode(data) // JSON string with type markers
const back = decode(str) // Map, Set, BigInt, Date all restored!
```

<Callout type="info">
To use devalue with the client, set the `Accept: application/x-devalue+json` header. The server responds with devalue-encoded JSON automatically.
</Callout>

## Size Comparison

| Payload | JSON | MessagePack | Savings |
|---|---|---|---|
| 1 object | 33 B | 22 B | **33%** |
| 10 objects | 661 B | 493 B | **25%** |
| 100 objects | 9.5 KB | 6.7 KB | **30%** |
