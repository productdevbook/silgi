Deploy your Silgi router as a serverless function on AWS Lambda. Works with API Gateway REST (v1) and HTTP (v2) event formats.

## Setup

```ts twoslash
// @noErrors
import { createHandler } from 'silgi/aws-lambda'
import { appRouter } from './router'

export const handler = createHandler(appRouter, {
  context: (event) => ({
    db: getDB(),
    userId: event.requestContext?.authorizer?.userId,
    ip: event.headers['x-forwarded-for'],
  }),
})
```

## API Gateway configuration

Map a catch-all route to your Lambda:

```
ANY /api/{proxy+}  →  your-lambda-function
```

Then set the prefix in the adapter:

```ts twoslash
// @noErrors
export const handler = createHandler(appRouter, {
  prefix: '/api',
  context: (event) => ({ db: getDB() }),
})
```

## Input parsing

- **POST/PUT/PATCH** — body is parsed as JSON
- **GET** — input from `?data=` query parameter
- Base64-encoded bodies (API Gateway binary mode) are decoded automatically

## Cold start optimization

Silgi compiles the router once at module load time. The compiled pipeline is reused across Lambda invocations — no per-request compilation overhead.

```ts twoslash
// @noErrors
// Router is compiled once, reused across invocations
import { appRouter } from "./router"
export const handler = createHandler(appRouter, { ... })
```

<Callout type='info'>
  For minimal cold starts, keep your Lambda package small. Silgi's single-package design helps — no 35+ scoped packages
  to load.
</Callout>
