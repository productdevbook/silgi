title: Astro
description: Use Silgi with Astro API routes.
---

[Astro](https://astro.build) supports server-side API routes via file-based routing. Silgi integrates via a catch-all endpoint.

## Setup

Create a catch-all API route:

```ts twoslash
// @noErrors
// src/pages/api/rpc/[...path].ts
import { createHandler } from 'silgi/astro'
import { appRouter } from '~/server/rpc'

const handler = createHandler(appRouter, {
  prefix: '/api/rpc',
  context: (ctx) => ({
    db: getDB(),
    headers: Object.fromEntries(ctx.request.headers),
  }),
})

export const GET = handler
export const POST = handler
```

Procedures are accessible at:

```
POST /api/rpc/users/list
POST /api/rpc/users/create
GET  /api/rpc/health
```

## SSR mode

Silgi requires server-side rendering. Enable it in your Astro config:

```ts twoslash
// @noErrors
// astro.config.mjs
export default defineConfig({
  output: 'server',
})
```

## Analytics and Scalar

Enable the built-in [analytics dashboard](/docs/analytics) or [Scalar API reference](/docs/scalar) directly in the adapter:

```ts twoslash
// @noErrors
const handler = createHandler(appRouter, {
  context: (ctx) => ({ headers: Object.fromEntries(ctx.request.headers) }),
  analytics: {
    auth: process.env.ANALYTICS_TOKEN,
  },
  scalar: true,
})
```
