## Error Maps

Define possible errors per procedure. The `fail()` function is typed from this map.

```ts twoslash
// @noErrors
const deleteUser = mutation({
  input: z.object({ id: z.number() }),
  errors: {
    NOT_FOUND: 404,                         // status shorthand
    FORBIDDEN: {                            // with typed data
      status: 403,
      data: z.object({ reason: z.string() }),
    },
  },
  resolve: ({ input, ctx, fail }) => {
    const user = ctx.db.users.get(input.id)
    if (!user) fail("NOT_FOUND")                              // [!code highlight]
    if (user.ownerId !== ctx.user.id)
      fail("FORBIDDEN", { reason: "Not the owner" })          // [!code highlight]
    return ctx.db.users.delete(input.id)
  },
})
```

<Callout type="info">
`fail()` is fully typed — it only accepts error codes defined in the `errors` map, and `data` is required only when the error defines a data schema.
</Callout>

## KatmanError

Throw errors manually anywhere:

```ts twoslash
// @noErrors
import { KatmanError } from "katman"

// Auto-maps common codes to HTTP status
throw new KatmanError("NOT_FOUND")           // → 404
throw new KatmanError("UNAUTHORIZED")        // → 401
throw new KatmanError("BAD_REQUEST")         // → 400

// Custom
throw new KatmanError("CUSTOM_ERROR", {
  status: 418,
  message: "I'm a teapot",
  data: { reason: "custom" },
})
```

## Client-Side Handling

<Tabs items={['try/catch', 'safe() wrapper']}>
  <Tab value="try/catch">
  ```ts twoslash
// @noErrors
  try {
    await client.users.delete({ id: 1 })
  } catch (error) {
    if (error instanceof KatmanError) {
      error.code   // "FORBIDDEN"
      error.status // 403
      error.data   // { reason: "Not the owner" }
    }
  }
  ```
  </Tab>
  <Tab value="safe() wrapper">
  ```ts twoslash
// @noErrors
  import { safe } from "katman/client"

  const { error, data, isError } = await safe(
    client.users.delete({ id: 1 })
  )

  if (isError) {
    console.log(error.code) // "FORBIDDEN"
  } else {
    console.log(data) // success result
  }
  ```
  </Tab>
</Tabs>
