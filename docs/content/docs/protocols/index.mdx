Katman uses content negotiation — the client sends an `Accept` header and the server responds in that format. No configuration needed.

| Format | Size | Rich Types | Binary | Best For |
|---|---|---|---|---|
| **JSON** | Baseline | No | No | Default, universal |
| **MessagePack** | 30% smaller | Date | Yes | Mobile, bandwidth |
| **devalue** | ~Same | All | No | Date, Map, Set, BigInt |
