[Nitro](https://nitro.build) is the server engine behind Nuxt and TanStack Start. Katman integrates with Nitro in two ways: as the **server framework** (replacing H3, like Hono does) or alongside **H3 file-system routes**.

## Direct mode (recommended)

Just like [Hono with Nitro](https://github.com/nitrojs/nitro/tree/main/examples/hono), export an object with a `fetch` method from `server.ts`. Nitro detects it and uses Katman as the server framework:

```ts twoslash
// @noErrors
// server.ts
import { katman, KatmanError } from 'katman'
import { z } from 'zod'

const k = katman({
  context: (req) => ({
    db: getDB(),
    headers: Object.fromEntries(req.headers),
  }),
})

const { query, mutation, guard, router, handler } = k

const health = query(() => ({ status: 'ok' }))

const listUsers = query(z.object({ limit: z.number().optional() }), ({ input, ctx }) =>
  ctx.db.users.findMany({ take: input.limit }),
)

const appRouter = router({ health, users: { list: listUsers } })

// Export with fetch — Nitro uses it directly
export default { fetch: handler(appRouter) }
```

No H3, no file-system routing, no `katmanNitro` adapter — Katman IS the framework. Full content negotiation (JSON, MessagePack, devalue) works out of the box.

```
POST /health          → { status: "ok" }
POST /users/list      → [{ id: 1, name: "Alice" }, ...]
```

<Callout>
  This is the same pattern Hono uses with Nitro. Nitro detects the `fetch` method on the default export and routes all
  requests through it.
</Callout>

## H3 mode

If you need Nitro's file-system routing or want to mix Katman with H3 routes, use the `katmanNitro` adapter:

```ts twoslash
// @noErrors
// server/routes/rpc/[...path].ts
import { katmanNitro } from 'katman/nitro'
import { appRouter } from '~/server/rpc'

export default katmanNitro(appRouter, {
  context: (event) => ({
    db: getDB(),
    user: event.context.auth,
  }),
})
```

Nitro maps `[...path]` to the procedure path:

```
POST /rpc/users/list     → users/list
POST /rpc/users/create   → users/create
```

### Context from Nitro middleware

```ts twoslash
// @noErrors
// server/middleware/auth.ts
import { defineMiddleware } from 'nitro/h3'

export default defineMiddleware((event) => {
  event.context.auth = verifyToken(event.req.headers.get('authorization'))
})
```

```ts twoslash
// @noErrors
// server/routes/rpc/[...path].ts
export default katmanNitro(appRouter, {
  context: (event) => ({
    user: event.context.auth,
  }),
})
```

## Which mode to choose?

| Scenario                                           | Mode                                |
| -------------------------------------------------- | ----------------------------------- |
| New project, Katman is the only framework          | **Direct** (`{ fetch: handler() }`) |
| Existing Nitro/Nuxt app, adding RPC                | **H3** (`katmanNitro()`)            |
| Need H3 middleware (auth, session, etc.)           | **H3**                              |
| Want full protocol support (msgpack, devalue, SSE) | **Direct**                          |

<Callout type='info'>
  Both modes target Nitro v3 (H3 v2). For Nitro v2 (Nuxt 3), use the [H3 adapter](/docs/integrations/h3) instead.
</Callout>
