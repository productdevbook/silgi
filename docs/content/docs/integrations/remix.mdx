---
title: Remix
description: Use Katman with Remix action/loader routes.
---

[Remix](https://remix.run) uses file-based routing with `action` and `loader` exports. Katman integrates via a catch-all splat route.

## Setup

Create a catch-all route:

```ts twoslash
// @noErrors
// app/routes/rpc.$.tsx
import { katmanRemix } from 'katman/remix'
import { appRouter } from '~/server/rpc'

const handler = katmanRemix(appRouter, {
  prefix: '/rpc',
  context: (args) => ({
    db: getDB(),
    headers: Object.fromEntries(args.request.headers),
  }),
})

export const action = handler
export const loader = handler
```

Procedures are accessible at:

```
POST /rpc/users/list
POST /rpc/users/create
GET  /rpc/health
```

## Context from Remix loaders

The `context` function receives the standard Remix `LoaderFunctionArgs` / `ActionFunctionArgs` object:

```ts twoslash
// @noErrors
katmanRemix(appRouter, {
  context: (args) => ({
    user: args.context.user, // from Remix server context
    params: args.params,
  }),
})
```
