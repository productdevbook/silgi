## Which integration do I need?

| Scenario                                      | Integration                                         |
| --------------------------------------------- | --------------------------------------------------- |
| You use Nitro v3                               | [Nitro](/docs/integrations/nitro)                   |
| You use Nuxt 4                                 | [Nuxt](/docs/integrations/nuxt)                     |
| You use Next.js                                | [Next.js](/docs/integrations/nextjs)                |
| You have an existing Fastify app               | [Fastify](/docs/integrations/fastify)               |
| You have an existing Express app               | [Express](/docs/integrations/express)               |
| You use SvelteKit                              | [SvelteKit](/docs/integrations/sveltekit)           |
| You use Remix                                  | [Remix](/docs/integrations/remix)                   |
| You use Astro                                  | [Astro](/docs/integrations/astro)                   |
| You use SolidStart                             | [SolidStart](/docs/integrations/solidstart)         |
| Deploying to AWS Lambda                        | [AWS Lambda](/docs/integrations/aws-lambda)         |
| Electron, Web Workers, browser extensions      | [Message Port](/docs/integrations/message-port)     |
| Database with Drizzle ORM                      | [Drizzle](/docs/integrations/drizzle)               |
| Authentication with Better Auth                | [Better Auth](/docs/integrations/better-auth)       |
| LLMs calling your API as tools                 | [AI SDK](/docs/integrations/ai-sdk)                 |
| React Query (or Vue/Solid/Svelte Query)        | [TanStack Query](/docs/integrations/tanstack-query) |
| Next.js or TanStack Start with server actions  | [React](/docs/integrations/react)                   |
| Consuming an external OpenAPI service          | [OpenAPI Client](/docs/integrations/openapi-client) |

For standalone servers (no existing framework), you don't need any integration — just use `s.serve()` or `s.handler()` from the [Server](/docs/server) page.
