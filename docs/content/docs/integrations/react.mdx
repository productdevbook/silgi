React Server Actions let you call server-side code directly from React components without building an API client. Katman makes this type-safe with `[error, data]` tuples instead of try/catch.

## Prerequisites

You need a katman router defined on the server side. If you haven't set one up yet, see the [Getting Started](/docs/getting-started) guide.

## createAction

Wraps a single procedure. Returns `[error, data]` instead of throwing.

```ts twoslash title="app/actions.ts"
// @noErrors
"use server"
import { createAction } from "katman/react"
import { appRouter } from "./router"

export const listUsers = createAction(appRouter.users.list)
export const createUser = createAction(appRouter.users.create)
```

```tsx title="app/page.tsx"
import { listUsers } from "./actions"

export default async function Page() {
  const [error, users] = await listUsers({ limit: 10 }) // [!code highlight]

  if (error) {
    return <p>Error: {error.message}</p>
  }

  return (
    <ul>
      {users.map(u => <li key={u.id}>{u.name}</li>)}
    </ul>
  )
}
```

## createActions

Convert an entire router at once — mirrors the router structure:

```ts twoslash
// @noErrors
import { createActions } from "katman/react"

const actions = createActions(appRouter)

// Same nesting as your router
const [err, users] = await actions.users.list({ limit: 10 })
const [err2, user] = await actions.users.create({ name: "Alice" })
```

## createFormAction

For HTML `<form>` submissions with FormData:

```ts twoslash title="app/actions.ts"
// @noErrors
"use server"
import { createFormAction } from "katman/react"

export const submitUser = createFormAction(appRouter.users.create)
```

```tsx
<form action={submitUser}>
  <input name="name" />
  <input name="email" type="email" />
  <button type="submit">Create</button>
</form>
```

Bracket notation is supported: `user[name]` becomes `{ user: { name: value } }`.

<Callout>
Next.js `redirect()` and `notFound()` errors, and TanStack Router navigation errors, are automatically rethrown — they won't be caught as `[error, data]`.
</Callout>
