Katman integrates with Next.js App Router via catch-all API routes. Full content negotiation (JSON, MessagePack, devalue) works out of the box.

## Setup

Create a catch-all API route:

```ts twoslash
// @noErrors
// app/api/rpc/[...path]/route.ts
import { katmanNextjs } from 'katman/nextjs'
import { appRouter } from '~/server/rpc'

const handler = katmanNextjs(appRouter, {
  prefix: '/api/rpc',
  context: (req) => ({
    db: getDB(),
    headers: Object.fromEntries(req.headers),
  }),
})

export { handler as GET, handler as POST }
```

Procedures are accessible at:

```
POST /api/rpc/users/list
POST /api/rpc/users/create
GET  /api/rpc/health
```

## Server Actions

For server-first React apps, use [React Server Actions](/docs/integrations/react) instead of API routes:

```ts twoslash
// @noErrors
// app/actions.ts
'use server'
import { createAction } from 'katman/react'

export const listUsers = createAction(appRouter.users.list)
```

## Edge Runtime

The handler uses standard Fetch API — works on both Node.js and Edge runtime:

```ts twoslash
// @noErrors
export const runtime = 'edge'
```

<Callout type='info'>
  Katman compiles the router once at module load time. Next.js caches module imports across requests, so the compilation
  cost is paid only once per cold start.
</Callout>
