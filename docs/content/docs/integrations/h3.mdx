[H3](https://github.com/unjs/h3) is a minimal HTTP framework used by Nitro and Nuxt. Katman's H3 adapter lets you mount RPC procedures alongside your existing routes.

## Setup

```ts twoslash
// @noErrors
import { H3 } from "h3"
import { katmanH3 } from "katman/h3"

const app = new H3()

// Existing routes
app.get("/api/health", () => ({ status: "ok" }))

// Katman RPC under /rpc prefix
app.all("/rpc/**", katmanH3(appRouter, {
  context: (event) => ({
    db: getDB(),
    user: event.context.user,
  }),
}))
```

Each procedure becomes available at its path under the prefix:

```
POST /rpc/users/list
POST /rpc/users/create
```

## Context from H3

The `context` function receives the H3 event. Use it to bridge H3's auth, session, or any middleware data into Katman's context:

```ts twoslash
// @noErrors
katmanH3(appRouter, {
  context: (event) => ({
    user: event.context.user,     // from H3 middleware
    ip: event.req.headers.get("x-forwarded-for"),
  }),
})
```

## Nitro / Nuxt server routes

In a Nitro or Nuxt project, create a catch-all server route:

```ts twoslash
// @noErrors
// server/routes/rpc/[...].ts
import { katmanH3 } from "katman/h3"
import { appRouter } from "~/server/rpc"

export default katmanH3(appRouter, {
  prefix: "/rpc",
  context: (event) => ({ db: getDB() }),
})
```

<Callout type="info">
This adapter supports H3 v2 (used in Nitro v3 and Nuxt 4). For H3 v1, the API is similar but uses `defineEventHandler` instead of `defineHandler`.
</Callout>
