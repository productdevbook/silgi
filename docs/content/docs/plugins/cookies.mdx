Lightweight cookie utilities for Katman handlers. No dependencies. Works with both `serve()` and `handler()`.

## Reading cookies

```ts twoslash
// @noErrors
import { getCookie, parseCookies } from 'katman/cookies'

const auth = k.guard((ctx) => {
  const token = getCookie(ctx.headers, 'session')
  if (!token) throw new KatmanError('UNAUTHORIZED')
  return { sessionToken: token }
})

// Or parse all cookies at once
const all = parseCookies(ctx.headers)
// { session: "abc123", theme: "dark" }
```

## Setting cookies

`setCookie()` returns a `Set-Cookie` header value. Use it with `handler()` or in a wrap that modifies response headers:

```ts twoslash
// @noErrors
import { setCookie } from 'katman/cookies'

const header = setCookie('session', token, {
  httpOnly: true,
  secure: true,
  sameSite: 'lax',
  maxAge: 60 * 60 * 24, // 1 day
  path: '/',
})
// "session=abc123; Path=/; Max-Age=86400; HttpOnly; Secure; SameSite=Lax"
```

## Deleting cookies

```ts twoslash
// @noErrors
import { deleteCookie } from 'katman/cookies'

const header = deleteCookie('session')
// "session=; Path=/; Max-Age=0; HttpOnly; SameSite=Lax"
```

## Options

| Option     | Type                          | Default              | Description                       |
| ---------- | ----------------------------- | -------------------- | --------------------------------- |
| `maxAge`   | `number`                      | —                    | Cookie expiry in seconds from now |
| `expires`  | `Date`                        | —                    | Absolute expiry date              |
| `path`     | `string`                      | `"/"`                | Cookie path                       |
| `domain`   | `string`                      | —                    | Cookie domain                     |
| `secure`   | `boolean`                     | `true` in production | HTTPS only                        |
| `httpOnly` | `boolean`                     | `true`               | Prevent JavaScript access         |
| `sameSite` | `"strict" \| "lax" \| "none"` | `"lax"`              | SameSite policy                   |

<Callout>
  Defaults are secure by design: `HttpOnly`, `SameSite=Lax`, and `Secure` in production. You only need to override when
  you have a specific reason.
</Callout>
