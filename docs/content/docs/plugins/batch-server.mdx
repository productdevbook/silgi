The batch handler processes multiple procedure calls in one HTTP round-trip. It pairs with the client-side `BatchLink` to reduce network overhead when your UI makes many calls at once.

## Setup

```ts twoslash
// @noErrors
import { createBatchHandler } from 'katman/plugins'

const batchHandler = createBatchHandler(appRouter, {
  context: (req) => ({ db: getDB() }),
  maxBatchSize: 20,
})
```

Mount alongside your normal handler ŌĆö for example at `/batch`:

```ts twoslash
// @noErrors
// Using handler()
const handle = k.handler(appRouter)

export default {
  async fetch(request: Request) {
    const url = new URL(request.url)
    if (url.pathname === '/batch') return batchHandler(request)
    return handle(request)
  },
}
```

## Protocol

The client sends a `POST` with a JSON array of calls:

```json
[
  { "path": "users/list", "input": { "limit": 10 } },
  { "path": "users/get", "input": { "id": 1 } }
]
```

The server returns a JSON array of results (same order):

```json
[{ "data": [{ "id": 1, "name": "Alice" }] }, { "data": { "id": 1, "name": "Alice", "email": "alice@example.com" } }]
```

If a call fails, its entry has an `error` instead of `data`:

```json
[
  { "data": [{ "id": 1, "name": "Alice" }] },
  { "error": { "code": "NOT_FOUND", "status": 404, "message": "User not found" } }
]
```

## Options

| Option         | Type            | Default    | Description                             |
| -------------- | --------------- | ---------- | --------------------------------------- |
| `context`      | `(req) => TCtx` | (required) | Context factory ŌĆö called once per batch |
| `maxBatchSize` | `number`        | `50`       | Maximum calls per batch                 |

All calls in a batch share the same context (computed once from the HTTP request). Each call runs the full compiled pipeline including guards, wraps, and validation.

<Callout type='info'>
  The batch handler executes all calls concurrently with `Promise.all`. If you need sequential execution, use separate
  requests or implement a custom batch handler.
</Callout>

## Client setup

Use `BatchLink` from the client plugins:

```ts twoslash
// @noErrors
import { BatchLink } from 'katman/client/plugins'

const link = new BatchLink({
  url: 'http://localhost:3000',
  batchEndpoint: '/batch',
  maxBatchSize: 10,
})
```
