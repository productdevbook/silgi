Parse multipart file uploads with size and MIME type validation. Files are available in the procedure context.

## Usage

```ts twoslash
// @noErrors
import { fileGuard } from 'katman/plugins'

const uploadAvatar = k.mutation({
  use: [
    fileGuard({
      maxFileSize: 5 * 1024 * 1024, // 5 MB
      allowedTypes: ['image/*'],
    }),
  ],
  resolve: async ({ ctx }) => {
    const file = ctx.file
    const buffer = await file.arrayBuffer()
    return { name: file.name, size: file.size, type: file.type }
  },
})
```

## Options

| Option         | Type       | Default            | Description                                       |
| -------------- | ---------- | ------------------ | ------------------------------------------------- |
| `maxFileSize`  | `number`   | `10485760` (10 MB) | Maximum file size in bytes                        |
| `allowedTypes` | `string[]` | all                | Allowed MIME types (supports `image/*` wildcards) |
| `maxFiles`     | `number`   | `1`                | Maximum number of files                           |
| `fieldName`    | `string`   | `"file"`           | Form field name                                   |

## Multiple files

Set `maxFiles` to allow multiple uploads. Files are available as `ctx.files`:

```ts twoslash
// @noErrors
const uploadPhotos = k.mutation({
  use: [fileGuard({ maxFiles: 10, allowedTypes: ['image/*'] })],
  resolve: async ({ ctx }) => {
    return ctx.files.map((f) => ({ name: f.name, size: f.size }))
  },
})
```

## Manual parsing

For custom file handling, use `parseMultipart()` directly:

```ts twoslash
// @noErrors
import { parseMultipart } from 'katman/plugins'

const upload = k.mutation({
  resolve: async ({ ctx }) => {
    const { files, fields } = await parseMultipart(ctx.__request)
    return { fileCount: files.length, caption: fields.caption }
  },
})
```

<Callout type='info'>
  The file guard validates before the procedure runs. If a file exceeds the size limit or has a disallowed MIME type,
  the request is rejected with 400 or 413 immediately.
</Callout>
