## CORS

Generate CORS headers for your middleware or response:

```ts
import { corsHeaders } from "katman/cors"

const headers = corsHeaders({
  origin: "https://app.example.com",
  credentials: true,
  methods: ["GET", "POST"],
  maxAge: 86400,
})
```

Dynamic origin:

```ts
corsHeaders({
  origin: (requestOrigin) => requestOrigin.endsWith(".example.com"),
})
```

## OpenTelemetry

Wrap middleware — each procedure call becomes a span:

```ts
import { otelWrap } from "katman/otel"
import { trace } from "@opentelemetry/api"

const tracing = otelWrap(trace.getTracer("my-service"))

const proc = query({
  use: [tracing], // [!code highlight]
  resolve: ({ ctx }) => ctx.db.find(),
})
```

<Callout type="info">
Zero OTel dependency in katman — we use a lightweight `Tracer` / `Span` interface. Bring your own `@opentelemetry/api`.
</Callout>

## Pino Logging

Hook-based structured logging:

```ts
import pino from "pino"
import { loggingHooks } from "katman/pino"

const k = katman({
  context: (req) => ({}),
  hooks: loggingHooks({ logger: pino() }), // [!code highlight]
})
```

## Rate Limiting

Guard middleware with pluggable backend:

<Tabs items={['In-memory', 'Custom (Redis)']}>
  <Tab value="In-memory">
  ```ts
  import { rateLimitGuard, MemoryRateLimiter } from "katman/ratelimit"

  const rateLimit = rateLimitGuard({
    limiter: new MemoryRateLimiter({ limit: 100, windowMs: 60_000 }),
    keyFn: (ctx) => ctx.ip ?? "anonymous", // [!code highlight]
  })

  const proc = query({
    use: [rateLimit],
    resolve: () => ({ ok: true }),
  })
  ```
  </Tab>
  <Tab value="Custom (Redis)">
  ```ts
  import type { RateLimiter } from "katman/ratelimit"

  class RedisRateLimiter implements RateLimiter {
    async limit(key: string) {
      return { success: true, limit: 100, remaining: 99, reset: Date.now() + 60_000 }
    }
  }

  const rateLimit = rateLimitGuard({
    limiter: new RedisRateLimiter(),
    keyFn: (ctx) => ctx.ip ?? "anonymous",
  })
  ```
  </Tab>
</Tabs>

When the limit is exceeded, a `TOO_MANY_REQUESTS` (429) error is thrown with `retryAfter` data.
