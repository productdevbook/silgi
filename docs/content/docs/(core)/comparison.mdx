## Benchmarks

All benchmarks: Apple M3 Max, sequential requests. Numbers are real — reproduce them on your hardware.

### RPC pipeline (no HTTP, pure execution)

Raw middleware pipeline overhead — how fast the framework processes a call after TCP is stripped.

| Scenario             | Silgi      | oRPC     | H3 v2    |
| -------------------- | ---------- | -------- | -------- |
| No middleware        | **111 ns** | 685 ns   | 2,025 ns |
| Zod validation       | **241 ns** | 804 ns   | 4,214 ns |
| 3 middleware + Zod   | **297 ns** | 1,718 ns | 3,954 ns |
| 5 middleware + Zod   | **413 ns** | 2,219 ns | 3,917 ns |

### RPC over HTTP (Node.js v24.11.0)

Real HTTP servers, real TCP connections. 3000 sequential requests.

| Scenario               | Silgi               | oRPC            | H3 v2           |
| ---------------------- | ------------------- | --------------- | --------------- |
| Simple (no middleware) | **79µs** (12,592/s) | 83µs (12,048/s) | 78µs (12,753/s) |
| Zod validation         | **86µs** (11,627/s) | 120µs (8,315/s) | 93µs (10,707/s) |
| Guard + Zod            | **79µs** (12,706/s) | 116µs (8,625/s) | 96µs (10,359/s) |

### Tail latency (p99, Guard + Zod)

| Framework  | avg      | p50      | p99       |
| ---------- | -------- | -------- | --------- |
| **Silgi**  | **79µs** | **70µs** | **148µs** |
| oRPC       | 116µs    | 103µs    | 223µs     |
| H3 v2      | 96µs     | 82µs     | 236µs     |

### HTTP frameworks — Node.js

Simple POST endpoint returning JSON. 3000 sequential requests. No validation, no middleware.

| Framework  | avg     | p50     | p95     | p99     | req/s     |
| ---------- | ------- | ------- | ------- | ------- | --------- |
| Fastify    | 100µs   | 86µs    | 135µs   | 207µs   | 9,956/s   |
| Express    | 109µs   | 96µs    | 150µs   | 212µs   | 9,165/s   |
| Hono       | 115µs   | 104µs   | 151µs   | 211µs   | 8,679/s   |
| Silgi      | 130µs   | 109µs   | 202µs   | 331µs   | 7,672/s   |

Silgi uses the Fetch API (`Request`/`Response`) while Fastify/Express use native Node.js `req`/`res` — this adds ~30µs fixed overhead that doesn't scale with middleware complexity.

### HTTP frameworks — Bun 1.3.11

Same endpoint, all frameworks use `Bun.serve()`.

| Framework  | avg     | p50     | p95     | p99     | req/s      |
| ---------- | ------- | ------- | ------- | ------- | ---------- |
| Hono       | 46µs    | 45µs    | 53µs    | 64µs    | 21,769/s   |
| Elysia     | 46µs    | 45µs    | 54µs    | 82µs    | 21,657/s   |
| Silgi      | 51µs    | 47µs    | 77µs    | 97µs    | 19,499/s   |

On Bun all three are within 10%.

### Router

JIT-compiled radix tree router vs rou3.

| Scenario                | Silgi       | rou3    |
| ----------------------- | ----------- | ------- |
| Static `/users/list`    | **3.1 ns**  | 3.9 ns  |
| Param `/users/123`      | **22.0 ns** | 25.6 ns |
| Wildcard `/files/a/b/c` | **19.2 ns** | 91.3 ns |
| Miss `/missing/deep`    | **4.5 ns**  | 22.3 ns |

### Memory

> 50K calls, 3 guards + Zod validation, `--expose-gc`

| Framework | Bytes/call |
| --------- | ---------- |
| Silgi     | ~40 bytes  |
| oRPC      | ~56 bytes  |

### What the benchmarks don't show

Real-world APIs spend 95%+ of time in DB queries and business logic. Pipeline overhead matters most for high-throughput, low-latency services. All benchmarks are sequential — concurrent load may show different characteristics.

## Choosing the right tool

Every framework in this comparison is well-built and actively maintained. The best choice depends on your project.

**oRPC** — broadest ecosystem (35+ packages), Cloudflare Durable Objects, dedicated RPC protocol, largest RPC community after tRPC.

**tRPC** — most mature type-safe RPC, largest community, extensive tutorials, proven production track record.

**Hono** — lightweight runtime-agnostic web framework. Node.js, Bun, Deno, Cloudflare Workers.

**Elysia** — best Bun performance with elegant TypeBox-powered API.

**Fastify** — mature Node.js framework with vast plugin ecosystem and enterprise adoption.

**Express** — maximum middleware compatibility and the widest community.

**Silgi** — compiled pipelines, single package, built-in analytics, Guard/Wrap model, client resilience, per-procedure OpenAPI.
