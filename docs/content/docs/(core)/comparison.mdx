## Architecture

### Guard / Wrap vs middleware chain

Most frameworks have one middleware type. Silgi has two:

- **Guard** — runs before, enriches context. No `next()`. Sync fast-path.
- **Wrap** — runs before AND after (onion). Has `next()`. For timing, caching, error capture.

```
Hono/Express:  middleware → middleware → middleware → handler
Elysia:        beforeHandle → handler → afterHandle
Fastify:       onRequest → preHandler → handler → onSend
oRPC/tRPC:     middleware → middleware → handler
Silgi:         guard → guard → [wrap → [wrap → handler]]
```

### Compiled pipeline

Silgi compiles the middleware chain once at startup — guards are unrolled (0-4 specialization, no loop), context pool eliminates per-request allocation. Elysia also compiles routes via Bun's JIT, but only runs on Bun. Silgi's compilation works on all runtimes.

### Content negotiation

**Silgi** inspects `Accept` header → responds in JSON, MessagePack, or devalue. Standard HTTP.

**Hono/Express/Fastify** return JSON by default. Content negotiation is manual.

**Elysia** returns JSON by default. TypeBox provides type coercion, not protocol negotiation.

## Benchmarks

All benchmarks: Apple M3 Max, Node v24.11.0, sequential requests.

### Pipeline performance (no HTTP)

Raw middleware pipeline overhead — how fast the framework processes a call after TCP is stripped.

| Scenario             | Silgi      | oRPC     | H3 v2    | vs oRPC         | vs H3            |
| -------------------- | ---------- | -------- | -------- | --------------- | ---------------- |
| No middleware        | **111 ns** | 685 ns   | 2,025 ns | **6.2x faster** | **18.2x faster** |
| Zod validation       | **241 ns** | 804 ns   | 4,214 ns | **3.3x faster** | **17.5x faster** |
| 3 middleware + Zod   | **297 ns** | 1,718 ns | 3,954 ns | **5.8x faster** | **13.3x faster** |
| 5 middleware + Zod   | **413 ns** | 2,219 ns | 3,917 ns | **5.4x faster** | **9.5x faster**  |

### HTTP performance (real TCP)

3000 sequential requests per scenario.

| Scenario               | Silgi               | oRPC            | H3 v2           |
| ---------------------- | ------------------- | --------------- | --------------- |
| Simple (no middleware) | **79µs** (12,592/s) | 83µs (12,048/s) | 78µs (12,753/s) |
| Zod validation         | **86µs** (11,627/s) | 120µs (8,315/s) | 93µs (10,707/s) |
| Guard + Zod            | **79µs** (12,706/s) | 116µs (8,625/s) | 96µs (10,359/s) |

For simple routes, all three are within 10%. With middleware, Silgi is **1.5x faster than oRPC**.

### Tail latency (p99)

| Framework  | avg      | p50      | p99       |
| ---------- | -------- | -------- | --------- |
| **Silgi**  | **79µs** | **70µs** | **148µs** |
| oRPC       | 116µs    | 103µs    | 223µs     |
| H3 v2      | 96µs     | 82µs     | 236µs     |

p99 is **34% lower** than oRPC — compiled pipelines keep tail latency predictable.

### Router performance

| Scenario                | Silgi       | rou3    |                  |
| ----------------------- | ----------- | ------- | ---------------- |
| Static `/users/list`    | **3.1 ns**  | 3.9 ns  | **1.23x faster** |
| Param `/users/123`      | **22.0 ns** | 25.6 ns | **1.16x faster** |
| Wildcard `/files/a/b/c` | **19.2 ns** | 91.3 ns | **4.75x faster** |
| Miss `/missing/deep`    | **4.5 ns**  | 22.3 ns | **4.96x faster** |

| **Silgi** | ~40 bytes  | **1x**    |

## Where oRPC is ahead

- **Ecosystem breadth** — 35 packages with more niche integrations
- **Cloudflare Durable Objects** — Hibernation + Durable Iterator
- **RPC protocol** — Native type serialization without header negotiation
- **Community size** — Larger user base, more battle-tested
- **Angular support** — TanStack Query for Angular

## Where Silgi is ahead

- **Single package** — one install, no version coordination
- **Compiled pipeline** — 3-6x faster than oRPC, 34% lower p99
- **Guard / Wrap model** — clearer separation of concerns
- **Typed guard errors** — auto-merge into procedures and OpenAPI spec
- **Built-in analytics** — live dashboard with session tracking and span tracing
- **Client resilience** — [retry, circuit breaker, timeout](/docs/client-plugins) built-in
- **createCaller** — [test without HTTP](/docs/testing), full pipeline execution
- **OpenAPI per-procedure** — [operationId, security, spec override](/docs/openapi)
- **Response caching** — `cacheQuery()` with TTL, SWR, pluggable backends
- **Content negotiation** — standard HTTP, works with any client
