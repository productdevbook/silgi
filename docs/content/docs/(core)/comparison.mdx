All benchmarks: Apple M3 Max, sequential requests. Numbers are real — reproduce them on your hardware.

## HTTP — Node.js v24.11.0

Simple POST endpoint returning JSON. 3000 sequential requests, 200 warmup. No validation, no middleware.

| Framework  | avg     | p50     | p95     | p99     | req/s     |
| ---------- | ------- | ------- | ------- | ------- | --------- |
| Fastify    | 100µs   | 86µs    | 135µs   | 207µs   | 9,956/s   |
| Express    | 109µs   | 96µs    | 150µs   | 212µs   | 9,165/s   |
| Hono       | 115µs   | 104µs   | 151µs   | 211µs   | 8,679/s   |
| Silgi      | 130µs   | 109µs   | 202µs   | 331µs   | 7,672/s   |

Silgi uses the Fetch API (`Request`/`Response`) while Fastify and Express use native Node.js `req`/`res`. This adds ~30µs fixed overhead per request. The gap closes as middleware complexity grows.

## HTTP — Bun 1.3.11

Same endpoint, all frameworks use `Bun.serve()`.

| Framework  | avg     | p50     | p95     | p99     | req/s      |
| ---------- | ------- | ------- | ------- | ------- | ---------- |
| Hono       | 46µs    | 45µs    | 53µs    | 64µs    | 21,769/s   |
| Elysia     | 46µs    | 45µs    | 54µs    | 82µs    | 21,657/s   |
| Silgi      | 51µs    | 47µs    | 77µs    | 97µs    | 19,499/s   |

On Bun all three are within 10%.

## RPC pipeline (no HTTP)

Measures middleware pipeline overhead in isolation — no TCP, no serialization. Only applicable to RPC frameworks with a compiled pipeline.

| Scenario             | Silgi      | oRPC     | H3 v2    |
| -------------------- | ---------- | -------- | -------- |
| No middleware        | **111 ns** | 685 ns   | 2,025 ns |
| Zod validation       | **241 ns** | 804 ns   | 4,214 ns |
| 3 middleware + Zod   | **297 ns** | 1,718 ns | 3,954 ns |
| 5 middleware + Zod   | **413 ns** | 2,219 ns | 3,917 ns |

<Callout>
  Hono, Elysia, Fastify, and Express are not included — they don't have an equivalent pipeline concept measurable in isolation. Their overhead is in the HTTP benchmarks above.
</Callout>

## Notes

- Real-world APIs spend 95%+ of time in DB queries. Framework overhead matters for high-throughput services.
- All benchmarks are sequential. Concurrent load may differ.
- Elysia is Bun-only — excluded from Node.js benchmarks.

</Callout>

## Choosing the right tool

Every framework here is well-built and actively maintained.

**oRPC** — broadest RPC ecosystem, Cloudflare Durable Objects, dedicated RPC protocol.

**tRPC** — most mature type-safe RPC, largest community, proven track record.

**Hono** — lightweight, runs everywhere: Node.js, Bun, Deno, Cloudflare Workers.

**Elysia** — best Bun performance, elegant TypeBox API.

**Fastify** — mature Node.js, vast plugin ecosystem.

**Express** — maximum middleware compatibility, widest community.

**Silgi** — compiled pipelines, single package, built-in analytics, Guard/Wrap model.
