title: Route Metadata
description: Extract and share route metadata between server and client without leaking server code.
---

When using `$route({ path })` for REST-style APIs, the client needs to know which HTTP method and path to use for each procedure. `minifyContractRouter()` extracts this metadata at build time — zero server code in the client bundle.

## The problem

```ts
// Server defines REST paths
const list = s.$route({ method: 'GET', path: '/api/users' }).$resolve(...)

// Client doesn't know about '/api/users' — it only sees the router tree
const users = await client.users.list() // → POST /users/list (wrong!)
```

## The solution

**Build-time:** Extract route metadata to a JSON file.

```ts title="scripts/extract-routes.ts"
import { minifyContractRouter } from 'silgi/contract'
import { appRouter } from './server/router'
import { writeFileSync } from 'fs'

const routes = minifyContractRouter(appRouter)
writeFileSync('./src/routes.json', JSON.stringify(routes))
```

**Client:** Import the JSON and pass it to `createLink`.

```ts title="client.ts"
import { createClient } from 'silgi/client'
import { createLink } from 'silgi/client/ofetch'
import type { AppRouter } from '~~/server/router' // type-only, erased at build
import routes from './routes.json'

const link = createLink({
  url: 'http://localhost:3000',
  routes,
})
const client = createClient<AppRouter>(link)

const users = await client.users.list() // → GET /api/users ✅
```

## `minifyContractRouter()`

Walks a router tree and returns a nested object with only `{ path, method }` per procedure:

```ts
import { minifyContractRouter } from 'silgi/contract'

const routes = minifyContractRouter(appRouter)
// {
//   users: {
//     list:   { path: '/api/users',     method: 'GET'    },
//     create: { path: '/api/users',     method: 'POST'   },
//     get:    { path: '/api/users/:id', method: 'GET'    },
//     delete: { path: '/api/users/:id', method: 'DELETE'  },
//   }
// }
```

Procedures without `$route({ path })` are excluded — they use the default RPC tree path.

## Automation

Add to your build pipeline:

```json title="package.json"
{
  "scripts": {
    "routes:extract": "tsx scripts/extract-routes.ts",
    "prebuild": "pnpm run routes:extract"
  }
}
```

<Callout type="info">
The JSON file should be committed to your repo. It changes only when you add/modify `$route({ path })` definitions.
</Callout>
